对话接口

1、对话接口

接口URL： http://ai.aepic.net:8000/v1/szy_chat

请求方式： POST

请求示例：

{
    "qtype":2,
    "knowid":"",
    "query": "石家庄今天天气如何",
    "history": [],
    "stream":true
}

参数说明：

参数值示例值参数类型是否必填参数描述
qtype0int否调用类型：0表示与llm直接对话或加入提示词对话，1表示与知识库对话，2表示意图识别对话，默认值为0
knowid"法律法规"字符串否知识库名称，默认值为”“
query"你是谁"字符串是对话内容、问题
history[["你好", "你好！......"]]列表是对话历史记录
streamtrue布尔否是否选用流式对话选项，默认为true

***注一：history可为空，为对话历史记录。每次对话会返回用户传入的历史记录和最新的聊天记录，如果需要传入历史记录，可直接提取对话返回的列表，设置传入聊天历史的条数作为历史记录参数。 ***

*** 注二：qtype为对话类型选项，默认为与大模型直接对话，当选择1时，与知识库对话，因为知识库管理接口正在开发，目前知识库用的之前的知识库，所以当填写knowid参数为除空字符串意外的任何字符串，都会走默认的知识库，后续添加完知识库管理接口即可与用户构建的知识库id对话。

query表示用户对话内容或问题
history表示本次对话的历史记录，
如果用户传入参数中有history，会一并返回给用户
finished流式接口结束标志，false表示没有结束，true表示本次对话结束。
