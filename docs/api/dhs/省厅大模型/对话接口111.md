# 对话接口111.md

## 1、对话接口

**接口URL：** [http://ai.aepic.net:8000/v1/szy_chat](http://ai.aepic.net:8000/v1/szy_chat)

**请求方式：**  POST

**请求示例**：

```
{
    "user_id":"111",
    "dialogue_id":"222",
    "qtype":1,
    "knowid":"1",
    "query": "你好",
    "history": [],
    "stream":true
}
```

```

```

**参数说明：**

| 参数值      | 示例值                   | 参数类型   | 是否必填 | 参数描述                                               |
|----------|-----------------------|--------|------|----------------------------------------------------|
| user_id  | "111"                 | string | 是    | 用户id，用于存储历史消息                                      |
| dialogue | "222"                 | string | 是    | 对话id，用于存储历史消息                                      |
| qtype    | 0                     | int    | 否    | 调用类型：0表示与llm直接对话或加入提示词对话，1表示与知识库对话，2表示意图识别对话，默认值为0 |
| knowid   | "法律法规"                | 字符串    | 否    | 知识库名称，默认值为”“                                       |
| query    | "你是谁"                 | 字符串    | 是    | 对话内容、问题                                            |
| history  | [["你好", "你好！......"]] | 列表     | 是    | 对话历史记录                                             |
| stream   | true                  | 布尔     | 否    | 是否选用流式对话选项，默认为true                                 |

<u>***注一：history可为空，为对话历史记录。每次对话会返回用户传入的历史记录和最新的聊天记录，如果需要传入历史记录，可直接提取对话返回的列表，设置传入聊天历史的条数作为历史记录参数。
***</u>

***<u>
注二：qtype为对话类型选项，默认为与大模型直接对话，当选择1时，与知识库对话，因为知识库管理接口正在开发，目前知识库用的之前的知识库，所以当填写knowid参数为除空字符串意外的任何字符串，都会走默认的知识库，后续添加完知识库管理接口即可与用户构建的知识库id对话。</u>
***
